Brandnew: Sales Dashboard

Brief

As part of the Startup Institute Program I had the opportunity to work with Brandnew, a startup that connects brands with influential users to deliver campaigns on Instagram and Pinterest. Alan Venegas and I were given the task of conceptualising and building a Sales Dashboard that would display KPI's per sales person in the company. My focus was to design and build the Front End of the Dashboard. The project was completed in 60 hours, spread over a 5 week period.

Process

Our first step was to really get to know our users, and determine their requirements. We did this by speaking with key team members at Brandnew and learning about systems already in place. I also investigated Front End frameworks that would support the project objectives.

Client Needs

  • Goal - Drive Sales and increase healthy competition
  • How - A ranking of Sales People based on set criteria
  • When - Monthly, Yearly, and Quarterly Data
  • Where - Displayed on big screen in the main office

Proposed Solution

  • A series of bar and donut graphs that display data
  • A hierarchical sizing structure of charts, based on importance of criteria
  • A name index that identifies each person by colour, which is used to display the persons ranking in each chart
  • Three screens that regularly toggle between monthly, quarterly and yearly time periods

Frameworks

  • Morris.js
  • Twitter Bootstrap
  • GitHub

I used this research to create a series of mockups using Balsamiq, which were presented and approved by Brandnew. Due to the short timeframe of the project, we dived straight into building the first iteration of our MVP and presented to Brandnew for usability testing. The feedback highlighted important criteria that needed to be added to the dashboard. It also allowed us to refine the hierarchy of the information so that everything displayed was relevant. We then created our final iteration based on the feedback. The objective of the design was to make it easy for the user to identify each Sales Person and subsequently recognise patterns and correlations between each graph.

Further Iterations

Based on the final evaluation from Brandnew the next stage of the project would be to create a desktop version of the dashboard incorporating the following features:


  • A user login
  • Date picker functionality that would allow the user to select their own timeframes
  • Functionality that would allow the user to only look at data for a selected Sales Person
